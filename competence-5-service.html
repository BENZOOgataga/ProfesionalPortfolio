Mettre à disposition des utilisateurs un service informatique

Compétence 5

Tests, déploiement et accompagnement

Ce bloc regroupe trois compétences liées à la mise en service d'une solution informatique : réalisation des tests d'intégration et d'acceptation, déploiement en production, et accompagnement des utilisateurs. Les réalisations s'appuient sur le hackathon BlackRaven Group et sur les projets d'infrastructure conduits à la SEMMARIS.

Contextes

Deux projets de référence

Hackathon - BlackRaven Group

Le hackathon a abouti au déploiement de l'ensemble des services de BlackRaven Group sur une infrastructure auto-hébergée sous Dokploy (Docker + Docker Swarm) : GLPI sur Proxmox, site vitrine sur Netlify (blackraven.fr), API jeu de piste et frontend consumer sur Dokploy. Chaque service a fait l'objet de tests et d'une documentation utilisateur.

Infrastructure : Dokploy (Docker + Docker Swarm). Services : GLPI (Proxmox, API v1), blackraven.fr (Netlify, domaine IONOS), API jeu de piste + frontend (Dokploy).

Alternance - SEMMARIS

À la SEMMARIS, les projets techniques hebdomadaires ont tous impliqué une phase de tests et de déploiement sur des serveurs Linux. Les travaux autour de Dokploy, Ansible, Kubernetes et Argo CD constituent les réalisations les plus significatives de ce bloc.

Technologies : Dokploy, Ansible, Kubernetes, Argo CD, Linux.

Activités

Compétences travaillées

C16 : Réaliser les tests d'intégration et d'acceptation

BlackRaven Group : Rédaction de scénarios de tests fonctionnels et techniques pour l'ensemble des services de BlackRaven Group : GLPI (création de ticket, affectation, changement de statut, clôture avec compte rendu), site vitrine (navigation desktop et mobile, formulaire de contact), application PHP/MySQL (authentification, lecture et écriture en base, gestion des erreurs). Mise en place d'un environnement de préproduction pour valider chaque livrable avant passage en production.

SEMMARIS : Tests réalisés avant chaque déploiement : validation des playbooks Ansible sur un environnement isolé avant application sur les serveurs cibles, vérification du comportement des conteneurs dans le cluster Kubernetes, contrôle de la synchronisation Argo CD entre le dépôt Git et l'état du cluster.

C17 : Déployer un service

BlackRaven Group : Déploiement complet de l'infrastructure BlackRaven Group : installation et configuration de GLPI sur un serveur Proxmox (Active Directory, DNS, DHCP, GLPI, Suricata, Grafana) avec base de données MySQL, basculement de l'API GLPI v2 vers l'API v1 pour résoudre une incompatibilité avec les jetons hexadécimaux, déploiement du site vitrine sur Netlify avec enregistrement du nom de domaine blackraven.fr sur IONOS (mise en ligne initiale via CloudFlare Workers), et déploiement de l'application de jeu de piste géolocalisé sur Dokploy : API REST (api.blackraven.fr) avec gestion des utilisateurs, moteur d'énigmes, intégration Mapbox et stockage base de données, accompagnée de son frontend de consommation (api-consumer.blackraven.fr).

SEMMARIS : Déploiement de Dokploy sur un serveur Linux de test, offrant une interface de gestion des conteneurs Docker comparable à un PaaS léger. Écriture et exécution de playbooks Ansible pour automatiser l'installation et la configuration de services sur les serveurs Linux. Mise en place d'un cluster Kubernetes et intégration d'Argo CD pour la livraison continue depuis des dépôts Git vers le cluster (approche GitOps).

C18 : Accompagner les utilisateurs dans la mise en place d'un service

BlackRaven Group : Rédaction d'un guide d'utilisation de GLPI à destination des utilisateurs de BlackRaven Group : soumettre un ticket, suivre l'avancement de sa demande, communiquer avec le support et clôturer une demande résolue. Documentation technique et procédures internes publiées sur Mintlify (docs.blackraven.fr), couvrant les processus de travail, les décisions horodatées et le cadre méthodologique OSINT de la structure. Intégration d'un formulaire de remontée de besoin d'aide ou d'amélioration dans le site vitrine, relié à la boîte mail du support.

SEMMARIS : Documentation technique rédigée dans Confluence pour les membres du service informatique, couvrant les procédures d'exploitation des outils déployés (Ansible, Dokploy). Assistance apportée aux collègues lors de la prise en main de nouveaux services ou outils introduits dans l'environnement de travail.
